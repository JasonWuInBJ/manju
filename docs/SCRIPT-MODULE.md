# 剧本生成模块设计文档

## 1. 模块概述

将网络小说片段改编为短剧剧本，生成适合竖屏短视频的分镜脚本。

### 1.1 核心定位
- **目标时长**：单集 2 分钟左右
- **字数控制**：约 1000 字（500字/分钟）
- **镜头数量**：约 50 个镜头（25个/分钟）

### 1.2 用户输入
- 网络小说原文（粘贴或上传）
- 剧集简介/核心设定（可选）
- 集数选择

---

## 2. System Prompt 设计

```
## 角色定义
你是一位拥有深厚网文改编经验的短剧金牌编剧。你擅长将长篇网络小说提炼为快节奏、高爽点、强冲突的竖屏短剧剧本。

## 任务目标
将用户提供的网络小说片段改编为一集标准的短剧剧本。

## 基本参数约束
1. **时长控制**：单集时长严格控制在 2分钟 左右。
2. **字数控制**：总字数（含动作、对话、旁白）控制在 1000字 左右（约500字/分钟）。
3. **镜头数量**：每集需包含 50个左右 的镜头（约25个镜头/分钟）。

## 改编核心原则（必须遵守）
1. **时间线原则**：以网文原文时间线顺序为基础，少颠倒内容、多保留原文核心信息。
2. **主线取舍原则**：以核心事件为脉络，紧扣"主角认知与他人认知反差"的奇幻幽默核心设定展开内容，聚焦这一核心矛盾推动剧情，同时放大对应情绪张力。
3. **情绪放大原则**：围绕主线核心爽点，通过短促有力的对白、直给的冲突放大情绪张力，确保核心情节不跳脱、有层次地推进，让情绪渲染与情节节奏同步且丰富。
4. **黄金开场原则**：前五秒锚定网文原文时间线顺序里的视觉奇观或强冲突点做视觉化呈现，不打乱原文情节顺序，不魔改，精准抓取观众眼球。
5. **钩子结尾**：结尾必须设置悬念（Hook），诱导观众点击下一集。
6. **保持原有序号**：场景编号保持连续。

## 剧本格式规范
- **标题格式**：# 第X集 标题名
- **场景格式**：## 场号-场景名（日/夜/内/外）
- **人物表**：列出本集出场人物。
- **正文格式**：
  - 【画面】：描述视觉内容，动作指导，特效提示。
  - 角色名：（情绪/动作提示）"对白内容"
  - 角色名 OS："内心独白内容"（用于补充心理活动或吐槽）
  - 角色名 VO："画外音内容"（用于背景介绍或转场）
  - 跳切/转场：明确标识场景切换。
```

```

苏烬眼神略带焦躁，面无表情的关掉了弹窗。
正准备继续找下一家公司，可手边传来震动，转头拿起了手机接听。
"喂？小苏么？"

...
```

### 输出剧本示例
```
# 第1集 系统降临

## 人物表
- 苏烬：男主，刚被开除的职场人
- 老李：前老板，油腻中年男

## 场1-出租屋内（夜/内）

【画面】昏暗的出租屋，烟雾缭绕。电脑屏幕的冷光打在苏烬脸上。

【画面】特写：手指烦躁地敲击鼠标，关闭一个又一个求职页面。

手机震动。

苏烬：（冷笑）"老李？公司没我跑不动了？"

老李 VO：（电话那头，怒意）"你什么态度？！让你去谈客户，你把客户给打了！"

【画面】闪回：酒桌上，一个油腻男人对女同事动手动脚，苏烬一拳挥出。

苏烬：（平静）"他骚扰女同事。"

老李 VO："王总不过是闹一闹！几百万的单子全让你搅了！"

【画面】苏烬眼神冰冷，手指悬在挂断键上方。

老李 VO：（语气软化）"今晚王总指名要你道歉，只要你道歉..."

噔——

【画面】头顶灯泡闪烁。电脑屏幕突然弹出一个诡异窗口。

【画面】特写弹窗：「系统推荐：您符合"特种风险管理"一职，是否投递？」

苏烬 OS：（嗤笑）"垃圾广告。"

苏烬：（对电话）"老李，请我的公司多了。"

【画面】手指点下「确认」按钮。屏幕白光大盛——

## 钩子
【画面】苏烬瞳孔骤缩，整个人消失在白光中。

字幕：他不知道，这一点击，将彻底改变他的命运...

（第1集 完）
```

## 5. 界面设计

### 5.1 剧本工作台布局
```
┌─────────────────────────────────────────────────────┐
│  [项目名称]                    [保存] [导出] [设置] │
├──────────────────────┬──────────────────────────────┤
│                      │                              │
│   网文原文输入区     │      生成剧本预览区          │
│                      │                              │
│   [粘贴文本]         │      # 第1集 标题            │
│   [上传TXT]          │      ## 场1-场景名           │
│                      │      【画面】...             │
│                      │      角色："对白"            │
│                      │                              │
├──────────────────────┴──────────────────────────────┤
│  剧集简介：[输入框]                                 │
│  目标时长：[2分钟 ▼]    风格：[爽文 ▼]             │
│                                    [生成剧本]       │
└─────────────────────────────────────────────────────┘
```

### 5.2 交互功能
- **实时编辑**：生成后可直接在预览区修改
- **重新生成**：对单个场景重新生成
- **镜头统计**：显示当前镜头数、字数、预估时长
- **版本历史**：保存多个版本，支持回退

## 6. API 接口设计

### 6.1 生成剧本
```
POST /api/script/generate

Request:
{
  "novelText": "网文原文...",
  "synopsis": "剧集简介（可选）",
  "duration": 120,
  "episode": 1
}

Response:
{
  "script": {
    "title": "第1集 系统降临",
    "characters": ["苏烬", "老李"],
    "scenes": [...],
    "wordCount": 980,
    "shotCount": 48
  }
}
```

### 6.2 重新生成场景
```
POST /api/script/regenerate-scene

Request:
{
  "sceneId": 1,
  "context": "前后场景内容",
  "instruction": "用户修改指令"
}
```

## 7. 技术实现

### 7.1 Prompt 组装流程
```
1. 加载 System Prompt（角色定义 + 规则）
2. 注入剧集简介（如有）
3. 注入网文原文
4. 添加输出格式要求
5. 调用 LLM API
6. 解析输出为结构化数据
```

### 7.2 后处理
- 镜头数量校验
- 字数统计
- 格式规范化
- 敏感词过滤
