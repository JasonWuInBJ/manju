# 分镜设计模块文档

## 1. 模块概述

将剧本转化为详细的分镜脚本，为后续图像生成和视频合成提供精确指导。

### 1.1 核心功能
- 根据剧本自动生成分镜设计
- 每个镜头包含完整的视觉指导
- 支持手动调整和重新生成

### 1.2 分镜要素
每个镜头需包含以下要素：
- **镜头语言**：景别、运镜方式
- **场景设计**：环境、道具、氛围
- **角色表现**：动作、表情、情绪
- **光影效果**：光源、明暗、色调
- **音效/配乐**：环境音、对白、BGM

## 2. System Prompt 设计

```
## 角色定义
你是一位资深的分镜设计师，擅长将剧本转化为电影级的分镜脚本。
你精通镜头语言、光影设计和视觉叙事，能够精准把控画面节奏和情绪张力。

## 任务目标
根据用户提供的剧本，为每个场景生成详细的分镜设计。

## 分镜设计规范
每个镜头必须包含以下五个要素：

1. **镜头语言**
   - 景别：特写/近景/中景/全景/远景
   - 运镜：定镜/推镜/拉镜/摇镜/跟镜/俯拍/仰拍

2. **场景设计**
   - 环境描述：空间、道具、背景细节
   - 氛围营造：天气、时间、整体感觉

3. **角色表现**
   - 动作指导：具体的肢体动作
   - 表情情绪：面部表情、眼神、情绪状态

4. **光影效果**
   - 光源方向与类型
   - 明暗对比与色调倾向

5. **音效设计**
   - 环境音与对白
   - 配乐风格建议
```

## 3. 输出格式示例

### 镜头一
**镜头语言**：中景镜头，缓慢推镜。

**场景设计**：房间昏暗，空气中弥漫着烟雾，光线从窗帘缝隙中透进来，形成一道道散乱的光束。电脑屏幕的冷光映在苏烬的脸上，桌面散乱，有烟灰缸、打火机和几张揉皱的纸。

**音效**：键盘敲击声，伴随手机震动的嗡嗡声。

**角色表现**：他懒散地接起电话，语气冷淡，表情毫无波澜。

**音效**：苏烬的台词："喂？"

**音效**：老李的台词："小苏啊，是我，老李……"

### 镜头五

**场景设计**：苏烬的脸部特写，背景昏暗，只有屏幕的光映在他的脸上。

**角色表现**：苏烬冷笑，嘴角微微上扬，眼中带着一丝嘲讽。

**光影效果**：屏幕的蓝光在他脸上形成明暗对比，表现他内心的冷漠与讽刺。

## 4. 数据结构设计

```typescript
interface Storyboard {
  episodeId: string;
  shots: Shot[];
}

interface Shot {
  id: number;
  camera: {
    type: 'close-up' | 'medium' | 'wide' | 'extreme-wide';
    movement: 'static' | 'push' | 'pull' | 'pan' | 'tilt';
    angle: 'eye-level' | 'high' | 'low';
  };
  scene: string;
  character: string;
  lighting: string;
  audio: {
    sfx: string;
    dialogue?: string;
    music?: string;
  };
}
```

```
├──────────────────────┬──────────────────────────────┤

```

### 5.2 单镜头编辑面板
- 镜头语言选择器（景别、运镜下拉框）
- 场景描述文本框
- 角色表现文本框
- 光影效果文本框
- 音效/对白输入框
- 重新生成按钮

### 5.3 用户微调交互

**编辑模式**：
- 点击任意镜头卡片进入编辑模式
- 所有文本字段均可直接修改
- 修改后自动保存，支持撤销/重做

**镜头操作**：
| 操作 | 说明 |
|------|------|
| 编辑 | 点击镜头卡片，修改任意字段 |
| 重新生成 | 对单个镜头重新 AI 生成 |
| 删除 | 删除当前镜头 |
| 插入 | 在当前位置插入新镜头 |
| 拖拽排序 | 拖动镜头卡片调整顺序 |
| 复制 | 复制当前镜头作为模板 |

**批量操作**：
- 全部重新生成：重新生成整个场景的分镜
- 批量调整风格：统一修改光影/色调描述
- 导出分镜表：导出为 PDF/Excel 格式

**智能辅助**：
- 输入修改指令，AI 自动调整（如"让光线更暗一些"）
- 风格一致性检查：提示与整体风格不符的镜头
- 镜头时长估算：根据内容自动估算每个镜头时长

**版本管理**：
- 自动保存历史版本
- 可查看修改记录
- 支持回退到任意历史版本

## 6. API 接口设计

### 6.1 生成分镜
```
POST /api/storyboard/generate

Request:
{
  "scriptId": "xxx",
  "sceneId": 1
}

Response:
{
  "shots": [
    {
      "id": 1,
      "camera": { "type": "medium", "movement": "push" },
      "scene": "房间昏暗，烟雾弥漫...",
      "character": "苏烬坐在电脑前...",
      "lighting": "屏幕蓝光与昏暗对比...",
      "audio": { "sfx": "键盘敲击声", "dialogue": "" }
    }
  ]
}
```

### 6.2 重新生成单个镜头
```
POST /api/storyboard/regenerate-shot

Request:
{
  "shotId": 1,
  "instruction": "增加更多光影细节"
}
```

### 6.3 更新镜头（用户编辑）
```
PUT /api/storyboard/shot/:id

Request:
{
  "camera": { "type": "close-up", "movement": "static" },
  "scene": "修改后的场景描述...",
  "character": "修改后的角色表现...",
  "lighting": "修改后的光影效果...",
  "audio": { "sfx": "修改后的音效", "dialogue": "对白" }
}
```

### 6.4 镜头排序
```
PUT /api/storyboard/reorder

Request:
{
  "storyboardId": "xxx",
  "shotIds": [3, 1, 2, 4]  // 新的顺序
}
```

### 6.5 插入/删除镜头
```
POST /api/storyboard/shot
DELETE /api/storyboard/shot/:id
```
