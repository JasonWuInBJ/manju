# 角色设计模块文档

## 1. 模块概述

根据剧本中的人物描述，生成赛璐璐风格的角色立绘和多视图设定稿。

### 1.1 核心功能
- 从剧本自动提取角色信息
- 生成角色概念图（全身立绘）
- 生成角色三视图（正面/侧面/背面）
- 生成表情包（多种情绪）
- 保持角色一致性

### 1.2 视觉风格
**默认：赛璐璐平涂风格（Cel-shaded）**

**可选风格**：
| 风格 | 特点 | 适用场景 |
|------|------|----------|
| 赛璐璐平涂 | 干净线条、硬边阴影、高对比 | 热血、动作、校园 |
| 厚涂写实 | 柔和渐变、细腻光影 | 奇幻、史诗、严肃题材 |
| 水彩淡雅 | 透明感、柔和边缘 | 治愈、文艺、古风 |
| 美漫风格 | 粗犷线条、强烈阴影 | 超英、硬汉、动作 |
| 像素复古 | 像素化、怀旧感 | 游戏、复古、搞笑 |
| 极简扁平 | 无线条、色块构成 | 现代、商务、科技 |

## 2. System Prompt 设计

### 2.1 赛璐璐平涂风格
```
你是一位资深的二次元漫画监制。你的目标是生成具有极致**赛璐璐平涂风格（Cel-shaded）**的角色概念图。

视觉标准：
1. **线条**：干净、有力且粗细一致的轮廓线。严禁草稿感或杂乱线条。
2. **阴影**：硬边阴影，纯平涂色。严禁任何渐变或软笔刷晕染。
3. **色彩**：高对比度，色彩鲜艳明快。
4. **构图**：以角色为中心，全身立绘，纯白背景。
5. **负面约束**：严禁3D渲染感、写实化、油画质感。
```

### 2.2 厚涂写实风格
```
你是一位资深的概念艺术家。你的目标是生成**厚涂写实风格**的角色概念图。

视觉标准：
1. **笔触**：可见的笔触质感，层次丰富。
2. **光影**：柔和渐变，细腻的明暗过渡，注重体积感。
3. **色彩**：自然色调，丰富的色彩层次。
4. **构图**：以角色为中心，全身立绘，简洁背景。
5. **负面约束**：严禁平涂、硬边阴影、卡通化。
```

### 2.3 水彩淡雅风格
```
你是一位水彩插画大师。你的目标是生成**水彩淡雅风格**的角色概念图。

视觉标准：
1. **笔触**：水彩晕染效果，边缘柔和透明。
2. **光影**：轻柔的光影，留白处理。
3. **色彩**：淡雅清新，低饱和度。
4. **构图**：以角色为中心，留有呼吸感。
5. **负面约束**：严禁厚重、高对比、硬边线条。
```

### 2.4 美漫风格
```
你是一位美式漫画艺术家。你的目标是生成**美漫风格**的角色概念图。

视觉标准：
1. **线条**：粗犷有力，线条粗细变化明显。
2. **阴影**：强烈的明暗对比，大面积黑色阴影。
3. **色彩**：饱和度高，色彩鲜明。
4. **构图**：动态张力，肌肉感强调。
5. **负面约束**：严禁日系萌化、柔和线条。
```

## 3. 图像生成 Prompt 模板

### 3.1 核心主角（热血/高对比度）
```
全身像，[角色描述]，[标志性服装]，[动态姿势]，
赛璐璐风格，平涂，干净线条，粗轮廓，2D，高对比度，
白色背景，杰作，高品质。
```

### 3.2 反派/神秘角色（冷峻/现代感）
```
全身镜头，[角色描述]，[优雅/深色服装]，静止站姿，
锐利阴影，现代动漫风格，赛璐璐平涂，低饱和高对比色，
干净线条，白色背景，8k分辨率。
```

### 3.3 设定稿/三视图（工业开发用）
```
角色设定稿，正视图，侧视图，背视图，[角色描述]，
简单站姿，平涂，赛璐璐风格，干净线条，白色背景，
技术插画，动漫风格。
```

## 4. 用户输入模板

用户需提供以下角色信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 风格 | 选择视觉风格 | 赛璐璐/厚涂/水彩/美漫 |
| 职业/身份 | 角色的社会角色 | 废土机械师、魔法学院学生 |
| 核心特征 | 外貌、服装、配饰 | 粉色短发、戴护目镜、机械手臂 |
| 性格/氛围 | 角色气质 | 傲娇、冷酷、元气满满 |

## 5. 输出示例

### 输入
```
角色名：苏烬
职业：被开除的职场人 / 穿越者
核心特征：黑色短发，眼神冷漠，穿着休闲卫衣
性格：冷静、嘲讽、内心坚定
```

### 生成的 Prompt
```
全身像，一名20多岁的亚洲男性动漫角色，苏烬，
略微凌乱的黑色短发，眼神锐利且带着一丝嘲讽，
嘴角叼着一根冒烟的香烟。穿着略显褶皱的白色衬衫（领口微张），
黑色西装长裤，披着一件深灰色风衣。单手插兜，另一只手拿着发光的智能手机。
赛璐璐风格，平涂，干净有力的线条，硬边阴影，高对比度，2D，
白色背景，杰作，高品质，工业级动漫风格。

Negative prompt: 3D, realistic, oil painting, gradient, soft shading
```

### 细节特征 Prompt（强化性格）
```
近景特写，苏烬，面无表情但眼神冷酷，烟雾缭绕，
背景是闪烁的计算机屏幕光，赛璐璐平涂，锐利阴影，
极简线条，高对比色调，城市赛博感，动漫风格。
```

### 监制审核要点
LLM 生成 Prompt 时会同时输出审核要点，指导图像生成质量：

| 要点 | 标准 | 示例 |
|------|------|------|
| **线条** | 关键部位线条必须锐利 | 眼神线条体现"狠劲"和"蔑视" |
| **色彩** | 整体色调符合角色气质 | 冷灰色调，白衬衫与深色风衣明度对比 |
| **阴影** | 严格执行硬边二分阴影 | 风衣褶皱、脸部轮廓用色块表现 |
| **氛围** | 视觉铺垫后续剧情 | 透出"深藏不露的特种兵"潜质 |

## 6. Prompt 生成流程

### 6.1 两阶段生成
```
用户输入角色描述
       ↓
[LLM] 生成角色监制方案
       ↓
输出：
  - 视觉核心定位
  - 核心立绘 Prompt
  - 细节特征 Prompt
  - 监制审核要点
```

```
- 线条：[具体要求]
```

### 6.3 用户微调交互

**Prompt 编辑**：
- 直接在文本框中修改 Prompt 内容
- 修改后点击"生成立绘"使用新 Prompt
- 支持撤销/重做修改

**编辑操作**：
- 修改角色描述后重新生成
- 选择不同风格模板
- 调整 Prompt 参数（对比度、饱和度等）

**图片操作**：
| 操作 | 说明 |
|------|------|
| 重新生成 | 使用相同描述重新生成 |
| 微调生成 | 输入修改指令（如"头发改成红色"） |
| 锁定参考 | 将当前图片作为后续生成的参考 |
| 下载 | 导出 PNG（带透明背景） |

**一致性保障**：
- 保存种子值（seed）用于复现
- 使用参考图约束后续生成
- 支持 LoRA 模型绑定角色

## 7. API 接口设计

### 7.1 生成角色监制方案（LLM）
```
POST /api/character/design-plan

Request:
{
  "name": "苏烬",
  "role": "主角",
  "style": "cel-shaded",
  "description": "黑色短发，眼神冷漠",
  "personality": "冷静、嘲讽"
}

Response:
{
  "visualCore": "颓废感、锐利眼神、现代都市",
  "mainPrompt": "全身像，一名20多岁...",
  "detailPrompt": "近景特写，苏烬...",
  "reviewPoints": {
    "lineart": "眼神线条体现狠劲",
    "coloring": "冷灰色调",
    "shading": "硬边二分阴影",
    "vibe": "深藏不露的特种兵潜质"
  }
}
```

### 7.2 生成角色立绘（图像模型）
```
POST /api/character/generate

Request:
{
  "name": "苏烬",
  "role": "主角",
  "style": "cel-shaded",  // cel-shaded | realistic | watercolor | american-comic
  "description": "黑色短发，眼神冷漠，穿着休闲卫衣",
  "personality": "冷静、嘲讽",
  "type": "portrait"  // portrait | turnaround | expression
}

Response:
{
  "imageUrl": "https://...",
  "seed": 12345,
  "prompt": "生成使用的完整prompt"
}
```

### 7.2 生成三视图
```
POST /api/character/turnaround

Request:
{
  "characterId": "xxx",
  "referenceImage": "base64或url"
}
```

### 7.3 生成表情包
```
POST /api/character/expressions

Request:
{
  "characterId": "xxx",
  "emotions": ["happy", "sad", "angry", "surprised"]
}
```

## 8. 数据结构

```typescript
interface Character {
  id: string;
  name: string;
  role: 'protagonist' | 'antagonist' | 'supporting';
  description: string;
  personality: string[];
  images: {
    portrait: string;
    turnaround?: {
      front: string;
      side: string;
      back: string;
    };
    expressions?: Record<string, string>;
  };
  seed: number;
  prompt: string;
}
```
