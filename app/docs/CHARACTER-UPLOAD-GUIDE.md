# 角色上传功能使用说明

## 功能概述

角色上传功能允许你将设计好的角色上传到 Sora 平台，获取角色 ID 后可用于后续的视频生成。整个流程分为两个步骤：

1. **生成角色视频** - 使用角色图片生成一段短视频
2. **上传角色** - 将视频上传到 Sora 平台获取角色 ID

---

## 使用流程

### 前置条件

在开始之前，请确保已完成以下步骤：

1. ✅ 创建项目
2. ✅ 生成剧本
3. ✅ 提取或创建角色
4. ✅ 生成角色绘图 Prompt
5. ✅ 生成角色图片

---

### 第一步：生成角色视频

#### 1. 进入角色设计页面

导航到：`项目详情 → 角色设计`

#### 2. 选择要上传的角色

在左侧角色列表中，点击选择需要上传的角色。

#### 3. 滚动到"角色上传"区域

在角色详情页面底部，找到"角色上传"模块。

#### 4. 配置视频参数

**第一步：生成角色视频**

- **视频提示词**（可选）
  - 默认值：`面对镜头，说天气不错，一起出去玩，和镜头互动；纯白色背景`
  - 可根据需要自定义角色的动作、表情等
  - 建议保持简洁明确

- **时长**
  - 可选：5秒 / 10秒 / 15秒
  - 默认：15秒
  - 建议：选择 15秒 以获得更完整的角色展示

- **比例**
  - 可选：1:1 / 16:9 / 9:16
  - 默认：9:16（竖屏）
  - 建议：根据后续使用场景选择

#### 5. 点击"生成视频"按钮

- 系统会显示"生成中..."状态
- 生成过程约需 1-3 分钟
- 可以切换到其他页面，回来后会自动继续轮询

#### 6. 预览视频

- 视频生成成功后会自动显示在界面上
- 点击播放按钮预览视频效果
- 确认角色展示效果是否满意

---

### 第二步：上传角色

#### 1. 确认视频已生成

确保"第一步：生成角色视频"区域显示了视频预览。

#### 2. 点击"上传角色"按钮

在"第二步：上传角色"区域，点击"上传角色"按钮。

#### 3. 等待上传完成

- 系统会显示"上传中..."状态
- 上传过程约需 30秒-2分钟
- 可以切换到其他页面，回来后会自动继续轮询

#### 4. 查看上传结果

上传成功后，界面会显示：

```
✓ 角色已上传成功

角色 ID:
anna-dian.noirgrit
```

- 绿色的"已上传"标签会显示在标题栏
- 角色 ID 会以等宽字体显示在绿色背景框中
- 角色列表中也会显示"已上传"标签

## 状态说明

### 视频生成状态

| 状态 | 说明 | 操作 |
|------|------|------|
| 未生成 | 尚未生成视频 | 点击"生成视频"按钮 |
| 生成中... | 视频正在生成 | 等待完成（可切换页面） |

| 生成失败 | 视频生成失败 | 查看错误信息，重新生成 |

### 角色上传状态

| 状态 | 说明 | 操作 |
|------|------|------|
| 未上传 | 尚未上传角色 | 点击"上传角色"按钮 |
| 上传中... | 角色正在上传 | 等待完成（可切换页面） |
| 已上传 | 上传成功，显示角色 ID | 可重新上传更新 ID |
| 上传失败 | 角色上传失败 | 查看错误信息，重新上传 |

## 常见问题

### Q1: 为什么"生成视频"按钮是灰色的？

**A:** 请确保已完成以下步骤：
- ✅ 已生成角色图片
- ✅ 角色图片 URL 存在

### Q2: 为什么"上传角色"按钮是灰色的？

**A:** 请确保已完成以下步骤：
- ✅ 已生成角色视频
- ✅ 视频预览正常显示

### Q3: 视频生成失败怎么办？

**A:** 可能的原因和解决方法：
1. **网络问题** - 检查网络连接，重试
2. **API 配额不足** - 检查 RunningHub API 余额
3. **图片 URL 无效** - 重新生成角色图片
4. **服务器错误** - 查看服务器日志，联系技术支持

### Q4: 角色上传失败怎么办？

**A:** 可能的原因和解决方法：
1. **视频 URL 无效** - 重新生成角色视频
2. **网络超时** - 重试上传
3. **API 错误** - 查看服务器日志，联系技术支持

### Q5: 可以重新上传角色吗？

**A:** 可以！点击"重新上传"按钮即可更新角色 ID。

### Q6: 切换页面后任务会中断吗？

**A:** 不会！系统会自动保存任务 ID，回到页面后会自动恢复轮询。

### Q7: 角色 ID 有什么用？

**A:** 角色 ID 用于后续的视频生成，可以让 AI 使用该角色生成新的视频内容。

### Q8: 一个角色可以生成多个视频吗？

**A:** 可以！但每次上传会覆盖之前的角色 ID。建议确认视频效果满意后再上传。

## 技术说明

### API 接口

#### 1. 图生视频接口

```
POST https://www.runninghub.cn/openapi/v2/rhart-video-s/image-to-video-pro

请求参数:
{
  "prompt": "视频提示词",
  "imageUrl": "角色图片 URL",
  "duration": "15",
  "aspectRatio": "9:16"
}

响应:
{
  "taskId": "任务 ID",
  "status": "QUEUED"
}
```

#### 2. 角色上传接口

```
POST https://www.runninghub.cn/openapi/v2/rhart-video-s/sora-upload-character

请求参数:
{
  "videoUrl": "角色视频 URL"
}

响应:
{
  "taskId": "任务 ID",
  "status": "QUEUED"
}
```

#### 3. 任务查询接口

```
POST https://www.runninghub.cn/openapi/v2/query

请求参数:
{
  "taskId": "任务 ID"
}

响应:
{
  "taskId": "任务 ID",
  "status": "SUCCESS",
  "results": [
    {
      "outputType": "text",
      "text": "角色 ID"
    }
  ]
}
```

### 数据存储

角色上传相关的数据存储在 `Character` 表中：

```typescript
{
  id: string                    // 角色 UUID
  name: string                  // 角色名称
  imageUrl: string              // 角色图片 URL
  characterVideoUrl: string     // 角色视频 URL
  characterVideoTaskId: string  // 视频生成任务 ID（用于恢复轮询）
  soraCharacterId: string       // Sora 角色 ID
}
```

## 最佳实践

### 1. 视频提示词建议

✅ **推荐的提示词格式：**
```
面对镜头，[动作描述]，[表情描述]，和镜头互动；纯白色背景
```

✅ **示例：**
- `面对镜头，微笑挥手，友好的表情，和镜头互动；纯白色背景`
- `面对镜头，点头致意，自信的表情，和镜头互动；纯白色背景`
- `面对镜头，说话交流，开心的表情，和镜头互动；纯白色背景`

❌ **避免：**
- 过于复杂的动作描述
- 多个场景切换
- 特效或后期处理要求

### 2. 视频参数选择

| 用途 | 推荐时长 | 推荐比例 |
|------|---------|---------|
| 角色展示 | 15秒 | 9:16（竖屏） |
| 快速预览 | 5秒 | 1:1（方形） |
| 横屏视频 | 10-15秒 | 16:9（横屏） |

### 3. 工作流程建议

1. **批量生成角色图片** - 先为所有角色生成图片
2. **逐个生成视频** - 按优先级生成角色视频
3. **预览确认** - 确认视频效果满意后再上传
4. **批量上传** - 集中上传所有角色
